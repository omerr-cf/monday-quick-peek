# Screenshots

This directory will contain screenshots of the extension for the Chrome Web Store listing and documentation.

## Required Screenshots

For Chrome Web Store submission, you'll need:

1. **1280x800px** - Main screenshot (required)
2. **640x400px** - Small screenshot (optional)
3. **440x280px** - Small tile (optional)

## Screenshot Guidelines

### Content to Include

- Tooltip showing notes preview
- Search functionality in action
- Settings popup
- Multiple notes displayed
- Error states (optional)

### Best Practices

- Use high-quality screenshots
- Show the extension in action
- Highlight key features
- Use Monday.com boards with realistic data
- Ensure text is readable
- Use consistent styling

## Screenshot Ideas

1. **Main Feature**: Tooltip showing notes preview on hover
2. **Search**: Tooltip with search functionality active
3. **Settings**: Settings popup with API key configuration
4. **Multiple Notes**: Tooltip with many notes displayed
5. **Empty State**: Tooltip showing "No notes available"
6. **Error State**: Error message in tooltip

## Tools for Screenshots

- **Chrome DevTools**: Use device toolbar for consistent sizing
- **Screenshot tools**:
  - macOS: Cmd+Shift+4 (select area)
  - Windows: Snipping Tool or Win+Shift+S
  - Linux: Various screenshot tools
- **Browser extensions**: Full Page Screen Capture, Awesome Screenshot

## Naming Convention

- `screenshot-main-1280x800.png` - Main screenshot
- `screenshot-search-1280x800.png` - Search feature
- `screenshot-settings-1280x800.png` - Settings popup
- `screenshot-notes-1280x800.png` - Multiple notes

## Tips

1. **Clean browser**: Close unnecessary tabs, use clean Monday.com board
2. **Consistent data**: Use the same test board for all screenshots
3. **Highlight features**: Use arrows or annotations to point out features
4. **Professional look**: Ensure Monday.com UI looks clean and professional
5. **Privacy**: Blur or remove any sensitive information

## After Taking Screenshots

1. Crop to required dimensions
2. Optimize file size (use tools like TinyPNG)
3. Ensure PNG format for best quality
4. Verify all text is readable
5. Check that extension features are clearly visible
